# Slishu — продуктовое и техническое ревью (2026-06-10)

Глубокое ревью через multi-agent workflow: parity-ресёрч актуального screenpipe (WebSearch),
аудит 4 слоёв кодовой базы, UX-аудит глазами нового пользователя, prod-readiness чеклист,
синтез 40 юзер-сторей + adversarial-верификация каждого статуса против реального кода.

---

## 1. Позиционирование: что изменилось у screenpipe и где мы

**screenpipe (июнь 2026, YC S26): open-core + подписка.** Core engine/CLI бесплатны (MIT), но вся
продуктовая поверхность — timeline, pipes, settings UI, AI chat — в подписанном desktop-приложении за
**$25/мес (Standard) / $50/seat (Pro, +cloud sync/AI) / $150/seat (Enterprise)**. Вход по email,
лицензия привязана к аккаунту. Cloud AI-пресет создаётся автоматически (дефолтный nudge в их облако),
хотя full-offline через Ollama возможен. Lifetime-лицензии отменены.

**Slishu-контрапункт (честный):** вся продуктовая поверхность — таймлайн, поиск, pipes, настройки —
локальна и бесплатна. Ноль аккаунтов, ноль egress, AI только через локальную Ollama.

**⚠️ Предостережение: старые аргументы устарели.** screenpipe теперь тоже AX-first с OCR-fallback и
event-driven (заявлено 5–10% CPU). Дифференциация «AX vs OCR 147%» съедена. **Живые дифференциаторы:**

| Дифференциатор | Статус |
|---|---|
| Ноль подписки / аккаунта / cloud-nudge | ✅ жив, теперь главный |
| Нативный SwiftUI vs Tauri/web | ✅ жив |
| Cross-lingual ru↔en semantic search (e5 + RRF) | ✅ жив — у screenpipe semantic слабый, cross-lingual не заявлен |
| On-device SFSpeech vs Whisper («главный пожиратель батареи» по их же докам) | ✅ жив |
| Разумный retention-дефолт (у них диск растёт бесконечно) | ✅ жив |
| AX-first vs OCR-first | ❌ съеден — у них тоже AX-first |

**Гэпы ожиданий класса продукта** (есть у screenpipe, нет у нас): диаризация спикеров, meetings как
сущность, PII-фильтр, ignored apps, bounding boxes/UI-tree для агентов, выбор аудио-устройств.

## 4. Главные технические находки (за пределами сторей)

**Ядро захвата** — самый большой разрыв план-vs-код:
- Режимы idle/active-text/burst-stream из плана НЕ реализованы — один фиксированный тик 3с.
- Per-app health (sick/slow/backoff) нет; capability-таблица in-memory, ocrOnly — навечно.
- cachedContent не инвалидируется при смене конфигурации дисплеев.

**Данные/поиск:**
- Миграция v3 дропнула старые векторы без re-embed джобы; нет учёта версии эмбеддера → следующая
  смена модели = снова drop-all.
- Embed-skip на дублях работает наполовину: основная ветка переэмбеддит неизменный текст.
- Эмбеддится только первый ~1800-символьный префикс — длинные экраны индексируются частично.
- dbBytes не включает размер самой БД (FTS+vec) в 20GB-бюджет.

**Известная боль подтверждена:** Keychain ACL ломается при ad-hoc ресайне (промпты + тихая регенерация
токена) — лечится Developer ID подписью + kSecUseDataProtectionKeychain.

## 5. Роадмап до первого продакшна

### v0.1 — «можно дать себе 24/7 и паре друзей» (все blockers + критичные must)

**Волна 1 — Доверие и lifecycle (без этого продукт врёт):**
1. **Упаковка**: Developer ID + Hardened Runtime + нотаризация + AppIcon + .dmg (US-01). Заодно чинит Keychain.
2. **Автостарт**: персист recordingEnabled + автозапись в bootstrap + SMAppService login item (US-04).
3. **Честные права**: notDetermined-детект, SCK-ошибки → .needsRestart + кнопка перезапуска + фоновый поллинг (US-03).
4. **Retention непрерывно**: таймер 30–60 мин + size-trigger + проверка свободного места перед writeFrame (US-36).
5. **Онбординг**: first-run sheet — consent «пишется всё» + степпер прав (US-02).
6. **Живой таймлайн**: ValueObservation/периодический refresh bounds+density (US-26).
7. **Индикатор по источникам** + surfacing смерти легов (US-05).
8. **e5 first-run**: download в Application Support, один общий кеш, UI-статус, retry; backfill кадров без вектора (US-24).
9. **os.Logger** по категориям + ротация server.log (US-07).

**Волна 2 — Обещание продукта (звонки + контроль данных):**
10. Semantic для аудио-транскриптов: vec_transcripts + embed в ingest + RRF (US-23).
11. Аудио в таймлайне: дорожка на скруббере, транскрипт-панель, AVPlayer m4a (US-28).
12. Мультимонитор минимум: дисплей сфокусированного окна + tie-break (ts,id) (US-11).
13. Устойчивость аудио к смене устройств (AirPods) + SCStream delegate (US-16).
14. Settings «Хранилище»: занято/retention-picker; удаление периода + «удалить всё» (US-35, US-37).
15. Разделить гейты: писать звук без speech-права (US-18).
16. Прогнать вживую: системный звук на реальном звонке + 8ч соак (закрывает known-unknowns).

### v0.2 — parity и polish (should)
- Фильтры поиска + пагинация сквозь SearchService → REST → MCP (US-21) + shard-фильтр vec (US-22).
- Pipe по расписанию + уведомление (US-33). Sparkle (US-06).
- Горячие клавиши + переход к дате (US-27). Ignored apps + пауза 15 мин (US-38). Экспорт (US-39).
- Idle-кадры вместо полного стопа + burst trio (US-10). VAD речь/музыка (US-17).
- Лейблы я/собеседник (US-19). REST: honest errors + транскрипт/аудио эндпоинты (US-29/31).
- Импорт из screenpipe (US-08) — приятный growth-ход, их юзеры сейчас злы на подписку.

### Сознательно НЕ делаем (позиция)
- Облако/аккаунты/телеметрию — никогда, это и есть продукт.
- Incognito-блэклист по умолчанию — дефолт «писать всё»; US-38 даёт opt-in исключения.
- Диаризацию полную (тяжёлая) — пока хватит mic/system-лейблов.
