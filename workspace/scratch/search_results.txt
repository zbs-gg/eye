# v3 Formula Simulation — Results

## 1. Methodology

**Mock event signatures** (`mock_event_signature`): keyword hits across 13 RU/EN emotion lexicons (стыд/shame, горд/pride, etc.), filtered by `sentiment` valence (sent≤-1 → negative classes only, sent≥+1 → positive only, 0 → both), augmented by `kind_tags → emotion` priors (`wound→grief`, `milestone→love`, `shame→shame`, …). Fallback: sentiment-only (≤-1→grief, ≥+1→love). Intensity = min(1, |sent|/2 + 0.2·user_flag + 0.1·(ntags−1)). Toward-entities extracted by name/keyword (anna/nik/elle/sonya/kristina/father/mother/mila).

**Mock query signature**: same lexicon on `user_query`, top-1 = primary, top-2 = secondary. No sentiment for queries, so valence-clash = `primary(q) positive ∧ dominant(e) negative`.

**Retrieval**: cosine over full 85 events via cached `text-embedding-3-large` vectors (`/tmp/pulse-v3-sim/vecs.npz`, 85×3072 and 47×3072). v2_pure = `cos × exp(-0.001·days)`. v3 = formula as specified. Recall@3 vs `ideal_top_3_event_ids` (embedding-derived GT).
