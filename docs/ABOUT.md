# Что такое ZBS Eye — полное описание

> Одной фразой: **ZBS Eye — это «вечная память» твоего Mac.** Он непрерывно и тихо записывает, что
> происходит на компьютере (экран + звук), осмысляет это и даёт найти, пересмотреть и проанализировать
> любой момент твоей цифровой жизни — **на 100% локально, без облака, без аккаунта, без подписки.**

Коднейм в коде — `ZBSEye` (типы, таргет, бинарь `ZBS Eye.app`). Снаружи везде — «ZBS Eye».

---

## 1. Зачем это вообще существует

Человек проводит за компьютером бóльшую часть дня, но почти ничего из этого не остаётся: вкладку
закрыл — мысль ушла, разговор в звонке — забылся, «где я это видел три недели назад» — не вспомнить.
Появилась категория продуктов «персональная память компьютера», но она осиротела и испортилась:
лидер ушёл к большой корпорации (фичи свернули), а оставшиеся аналоги — на **подписке $25–50/мес** и
с **обязательным облаком**, куда уезжает самое личное, что у тебя есть, — история того, что ты делаешь.

ZBS Eye занимает эту нишу с противоположной позицией: **всё остаётся у тебя.** Лёгкая нативная
альтернатива, которая не уходит в облако никогда — потому что история активности слишком личная, чтобы
её куда-то отдавать. Это не временная маркетинговая позиция, а архитектурный принцип (см. §4).

**Две продуктовые цели:**
1. **Память постфактум** — записать всё, найти по-человечески, пересмотреть, осмыслить. Это ядро (есть сейчас).
2. **Живой суфлёр** — подсказки в реальном времени поверх звонка/работы. Слой будущего поверх готового пайплайна.

## 2. Что он делает (возможности)

### Захват
- **Экран** → accessibility-текст (точный и лёгкий для батареи) + OCR там, где AX недоступен (GPU-рендереры,
  canvas, часть Electron); кадры в HEIC с perceptual-hash дедупом (одинаковые экраны не дублируются).
  Adaptive per-app: AX там, где приложение отдаёт семантику, OCR — где нет; решается в рантайме по качеству контента.
- **Аудио** → системный звук (звонки, встречи, видео) и микрофон → транскрипция **на устройстве** (SFSpeech),
  с VAD (не транскрайбим тишину/музыку).

### Осмысление (не просто сырьё, а структура)
- **Сцены / «День в активностях»** — кадры группируются в **сцены-активности**: «VS Code, 14:00–14:25,
  правил AXReader» вместо тысячи отдельных кадров. Видно, как день разбит на блоки. Вместо сырого OCR-дампа
  справа — **чистое саммари сцены** (приложение, окно/URL, ключевые темы; LLM-улучшение опционально).
- **Картограф** — дневной ИИ-инсайт: локальная LLM смотрит активность за день (топ-приложения, переключения
  контекста, темы) и даёт **2-3 конкретных наблюдения/совета** (self-improvement). Только on-device. В будущем —
  единый контур с Pulse/Atlas.

### Поиск и навигация
- **Гибрид-поиск** — полнотекстовый (FTS5) + семантический (multilingual-e5, 384-dim) через RRF.
  **Cross-lingual**: ищешь по-русски — находит английское (и наоборот).
- **Таймлайн** — перемотка по времени, плотность активности, плеер 1×/2×/4×, zoom день/час/10 мин, отдача
  кадров картинкой. Плавный: crossfade кадров, мягкий скруббер, микроанимации (с уважением к Reduce Motion).
- **«Спроси»** — задаёшь вопрос своей памяти → гибрид-поиск находит фрагменты → **локальная LLM** отвечает
  с цитатами-ссылками (кликаешь → прыжок на таймлайн). Локальный аналог «Ask Rewind». Модель выбирается из
  реально загруженных в **LM Studio / Ollama** (список тянется из `/v1/models`).

### Награды и прогресс
- **Геймификация** — стрики дней, вехи (1k/5k/10k/… кадров), «возраст памяти», прогресс к следующей вехе;
  на достижение вехи — тонкая визуальная награда (aurora-перелив). Чем дольше пользуешься — тем богаче память.

### Доступ для ИИ-агентов
- **Локальный REST** (127.0.0.1, Bearer-токен на всё кроме `/health`) + **MCP** (stdio) — чтобы LLM/агенты
  (Claude Desktop, Cursor) работали с твоей памятью как с инструментом. Ноль egress.

### Хранилище и данные
- **Retention по умолчанию ВЕЧНО** (явный выбор — иначе дни/размер).
- **Перенос на внешний SSD** одной кнопкой (relocatable; живая БД переносится online-бэкапом, без потери кадров).
- **Автобэкап в iCloud** — сжатым снапшотом (живую SQLite в iCloud класть нельзя — corruption).
- **Импорт прежней истории** (напр. из ~/.screenpipe) — забери накопленное себе.
- **Автоматизации** — daily-summary в файл/Obsidian; экспорт дня/всего.
- **Приватность** — пауза по приложению, исключения, удаление по времени; приложение не пишет себя.

## 3. Принципы (не нарушаются никогда)

- **Ноль egress.** Сервер слушает только `127.0.0.1`; всё кроме `/health` — за Bearer-токеном (в Keychain).
- **Ноль аккаунтов, ноль подписки, ноль телеметрии.** Это и есть продукт.
- **ИИ — только локальный** (LM Studio / Ollama / mlx_lm.server на localhost). Никаких облачных пресетов.
- **Дефолт — писать всё**, но человек главный: пауза, исключения приложений, удаление периода.
- **Нативность и лёгкость.** Swift/SwiftUI, аппаратное ускорение Apple Silicon, минимум зависимостей —
  в противовес тяжёлым web-обёрткам (Electron/Tauri).

## 4. Как это устроено (архитектура и стек)

- **Платформа:** Swift 6 (strict concurrency = `complete`), SwiftUI, target macOS 15+, Apple Silicon. ~10k строк Swift.
- **Конкурентность:** `@MainActor @Observable` сторы для UI; акторы для данных/захвата; один логический writer
  (`IngestService`); не-Sendable (`CVPixelBuffer`/`AXUIElement`/…) живут и умирают внутри одного актора.
- **Хранилище:** GRDB (`DatabasePool` + WAL) + **FTS5** (external-content) + **sqlite-vec** (статически
  слинкован — нотаризация без loadable-extension). Единый резолвер пути `StorageLocation` (relocate-aware).
- **Захват:** ScreenCaptureKit (HEIC, perceptual-hash), Accessibility API на выделенном потоке (cross-app,
  per-PID health), Vision OCR (ru+en, ANE). AXReader НЕ инспектирует собственный процесс (иначе self-AX
  reentrancy роняет SwiftUI на чужом потоке — реальный баг, исправлен).
- **Аудио:** CoreAudio process-tap (система) + микрофон, VAD-сегментер, SFSpeech on-device.
- **Поиск:** SearchService (FTS+vector RRF), EmbeddingService (multilingual-e5, **mean-pooling** — не CLS,
  иначе ×3 хуже на cross-lingual), temporal shards.
- **Сервер:** FlyingFox REST (`/v1`, 127.0.0.1, Bearer, path-traversal hardening) + MCP swift-sdk (stdio).
- **Безопасность пакета:** **Hardened Runtime БЕЗ App Sandbox** (sandbox несовместим с cross-app AX +
  локальным сервером). Минимальные entitlements.

Один бинарь — несколько режимов: GUI, `--mcp`, `--import-history`, `--relocate`, `--backup-now`, `--backup-verify`.

## 5. Раздача — Developer ID + нотаризация (НЕ App Store)

App Store требует App Sandbox, под которым **невозможен cross-app Accessibility** (ядро извлечения текста),
а профиль «вечная память, пишет всё» почти гарантированно реджектится по приватности. Поэтому, как
Rewind/screenpipe, ZBS Eye раздаётся **вне App Store** — нотаризованным Developer ID-билдом (`scripts/build-notarized.sh`):
запускается двойным кликом без «Open Anyway», подпись стабильна (права переживают обновления). Сетап — `docs/NOTARIZE.md`.

## 6. Место в экосистеме Garden

ZBS Eye живёт в `~/dev/ai/Garden/eye` рядом с другими продуктами семьи (Atlas, Cartographer, Pulse,
Garden-app) — каждый отдельный репо (`zbs-gg/eye`). Сейчас работает автономно и локально; в дальнейшем
«Картограф» соединяется с **Pulse / Atlas** в единый контур осмысления (память Mac → инсайты → действия),
оставаясь верным принципу «всё на устройстве».

## 7. Статус

**Работает и проверено вживую:** захват (экран + аудио), гибрид-поиск (cross-lingual), таймлайн (плавный),
сцены/«День в активностях», «Спроси» (RAG по локальной LLM с выбором модели из LM Studio), Картограф
(дневные инсайты), прогресс/вехи, REST + MCP, импорт истории, retention (вечно), relocatable хранилище,
iCloud-бэкап, daily-summary, экспорт. Нотаризованный Developer ID-релиз — есть.

**Отложено:** тест-таргет (XCTest), Sparkle-автообновления, глубокая интеграция Картографа с Pulse/Atlas,
v1.5 «живой суфлёр» (overlay в реальном времени).

Стратегия и приоритеты — в [`ROADMAP.md`](../ROADMAP.md). Архитектура и гид для контрибьюторов — в [`AGENTS.md`](../AGENTS.md).
